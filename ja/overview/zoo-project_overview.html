Zoo Project¶

WPS Server¶

ZOOプロジェクトは、Web Processing Servicesを作成およびチェーンするための開発者向けのWeb Processing Service(WPS)フレームワークを提供します。WPSは空間アルゴリズムを実行する関数へのWebアクセスを提供します。ZOO-Projectは多くのプログラミング言語をサポートしており、3つのデモアプリケーションが用意されています。第1に、GEOS <geos_overview> および OGR ライブラリに基づくシンプルなspatialtoolsサービスを使用します。第2に、VoronoiとDelaunayの三角分割(CGALライブラリから)を紹介します。第3に、WPSサービスとして動作するオー・ティー・ビーアプリケーションとの相互運用方法を紹介します。

3つのデモアプリケーションは、以下に示すZOO-Projectのデモランディングページから利用できます。

ZOOは3つの部分で構成されています:

ZOOカーネル : 異なるプログラミング言語でコーディングされたWebサービスの管理とチェーンを可能にする、強力なサーバーサイドCカーネルです。

ZOO Services : さまざまなオープンソースライブラリに基づく、サンプルWebサービスのスイートが増えています。

ZOO API : ZOOサービスを呼び出してチェーン接続できるサーバー側のJavaScript APIです。これにより、開発とチェーン接続のプロセスが容易になります。

ZOOは、ZOOのコアシステム(ZOO カーネル)を構成する’WPS Service Kernel’に基づいています。後者は、動的ライブラリをロードして、オンデマンドのWebサービスとして処理することができます。ZOOカーネルはC言語で書かれていますが、数多くのライブラリに接続するためにいくつかの一般的なプログラミング言語をサポートしています。特に、Webサービスのエンド開発者の仕事である単純化に接続します。

ZOOサービスは、メタデータファイル(.zcfg)と対応する実装のコードで構成されるリンクです。メタデータファイルには、WPS Exec Requestを使用して呼び出すことができるすべての関数と、目的の入出力が記述されています。サービスにはアルゴリズムと関数が含まれており、C/C++、Fortran、Java、Python、PHP、Ruby、C#、およびJavaScriptで実装できるようになりました。

主要機能¶

  • 多言語WPSサーバ
  • シンプルなWebサービスの作成と導入手順
  • ZOO APIによるシンプルなWPSチェーニング

実装されている標準規格¶

  • OGC WPS 1.0.0
  • OGC WPS 2.0.0
